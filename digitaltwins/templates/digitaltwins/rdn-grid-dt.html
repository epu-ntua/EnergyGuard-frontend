The RDN node provides a Digital Twin of a large-scale electricity grid with a high share of renewables, capable of RMS and EMT simulations as well as hardware-in-the-loop simulation with IEDs. Configure a use case, a grid section, and power setpoints for each asset, and the Digital Twin returns instantaneous voltage, current and frequency curves for every bus in the selected section.

Configure a request
One simulation, full grid dynamics

The Digital Twin runs your configured use case, grid section and power setpoints through EMT and RMS models, returning instantaneous voltage, current and frequency curves for every bus in the selected section.

How it works

Select a Use Case

Choose an automatic-interaction use case, or continue a previous request.

Configure

Pick a grid section and define each asset's power setpoints over time.

Simulate

The Digital Twin runs the grid scenario for the selected section.

Results

Explore per-bus voltage, current and frequency curves, or export them.

What you'll need to provide

Use case & grid section

Pick one of the automatic-interaction use cases (market results validation, voltage control, governor and load-frequency control), and the grid section to simulate — one of the Digital Twin's 7 grid sections, or the full grid.

Assets

Each asset connected to the grid section, identified by its bus type (PV, Wind, Small Hydro, Biomass, Large Hydro, Gas or Load).

Power setpoints

Active/reactive power (MW/MVAr) for each asset at up to 100 shared setpoint timestamps. The platform derives the exact timestamps for you from a start time, resolution and count.

Configuration

1. Use case

Select the use case to configure. Automatic-interaction use cases run immediately; the others require direct contact with the R&D Nester team.

2. Grid section

Focus on one grid area to reduce grid size and speed up the experiment, or use the full grid.

3. Setpoint timestamps

Every asset shares the same setpoint timestamps. Choose a start time, a resolution, and how many setpoints to submit (max {{ max_setpoints }}).
